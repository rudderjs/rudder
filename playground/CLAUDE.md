# Boost MCP Tools

When you need information about this project, **prefer Boost MCP tools
over re-deriving from the filesystem.** Tools are exposed by the
`rudderjs-boost` MCP server (registered in `.mcp.json`).

- `app_info` — installed `@rudderjs/*` versions, Node version, package manager
- `db_schema` — parsed Prisma schema models + raw `.prisma` source
- `route_list` — registered HTTP routes (method, path, middleware, source file)
- `model_list` — ORM models in `app/Models/` with tables + fields
- `config_get` — read `config/*.ts` files by key
- `db_query` — read-only `SELECT` queries against the project DB
- `last_error` — most recent entries from `storage/logs/`
- `read_logs` — log entries filtered by level + search term
- `browser_logs` — browser console logs from the Vite dev server
- `get_absolute_url` — convert a relative path to an absolute URL using `APP_URL`
- `search_docs` — keyword search across installed `@rudderjs/*` package docs
- `commands_list` — all `rudder` commands (built-in + package + user)
- `command_run` — execute a `rudder` command and capture stdout/stderr/exit

Per-package guidelines are also available as MCP resources:
`guidelines://all` (concatenated) or `guidelines://<package-short-name>`
(e.g. `guidelines://orm`).

=== skills activation ===

# Skills Activation

Skills live in `.ai/skills/`. **Load a skill only when its trigger
matches** — they're scoped knowledge modules, not always-on context.

- `ai-agents` — **ACTIVATE when:** building an `Agent` class, calling `Agent.prompt()`/`.stream()`, defining `tools()` or `middleware()`, or wiring conversations / failover. **SKIP when:** writing a tool definition by itself — load `ai-tools` instead.
- `ai-tools` — **ACTIVATE when:** writing a `toolDefinition()`, defining server or client tools, adding streaming yields, or wiring approval gates. **SKIP when:** configuring an `Agent` class itself — load `ai-agents` instead.
- `auth-setup` — **ACTIVATE when:** configuring guards/providers in `config/auth.ts`, vendoring auth views, wiring `Gate`/`Policy`, or working with password reset / email verification. **SKIP when:** a route handler that just reads `Auth.user()` — no setup needed.
- `mcp-servers` — **ACTIVATE when:** building an MCP server, using `@McpServer` / `@Tool` decorators, exposing tools or resources, or choosing HTTP vs stdio transport. **SKIP when:** consuming an external MCP server from an Agent — use `mcpClientTools()` and load `ai-tools` instead.
- `orm-models` — **ACTIVATE when:** creating or editing a `Model` class under `app/Models/`, writing queries / relationships, defining casts / accessors / factories, or building `JsonResource` API resources. **SKIP when:** a route handler that only reads a model — controller-views is enough.
- `controller-views` — **ACTIVATE when:** creating files in `app/Views/`, returning `view('id', props)` from a route handler, or overriding the view-id → URL mapping. **SKIP when:** working with Vike `+Page.tsx` / `+config.ts` directly — that's outside `app/Views/`.
