# S3 / R2 / MinIO

S3-compatible storage adapter for `@rudderjs/storage`.

## Installation

```bash
pnpm add @aws-sdk/client-s3
```

## Setup

Add an S3 disk to your storage configuration:

```ts
// config/storage.ts
import type { StorageConfig } from '@rudderjs/storage'

export default {
  default: Env.get('STORAGE_DRIVER', 's3'),
  disks: {
    local: {
      driver: 'local',
      root: './storage/app',
    },
    s3: {
      driver: 's3',
      bucket: Env.get('AWS_BUCKET'),
      region: Env.get('AWS_REGION', 'us-east-1'),
      accessKeyId: Env.get('AWS_ACCESS_KEY_ID'),
      secretAccessKey: Env.get('AWS_SECRET_ACCESS_KEY'),
      baseUrl: Env.get('AWS_URL'),
    },
  },
} satisfies StorageConfig
```

No changes are needed in `bootstrap/providers.ts` — `@rudderjs/storage` dynamically loads the `s3` driver when it sees `driver: 's3'` in a disk config.

## Configuration

### `S3DiskConfig`

| Option             | Type      | Description                                                                       |
|--------------------|-----------|-----------------------------------------------------------------------------------|
| `driver`           | `'s3'`    | Must be `'s3'` to select this adapter.                                            |
| `bucket`           | `string`  | Name of the S3 bucket.                                                            |
| `region`           | `string?` | AWS region (e.g. `'us-east-1'`). Required for AWS S3; may be omitted for R2/MinIO. |
| `endpoint`         | `string?` | Custom endpoint URL. Required for R2, MinIO, and other S3-compatible services.    |
| `accessKeyId`      | `string?` | AWS access key ID. Falls back to `AWS_ACCESS_KEY_ID` environment variable.        |
| `secretAccessKey`  | `string?` | AWS secret access key. Falls back to `AWS_SECRET_ACCESS_KEY` environment variable. |
| `baseUrl`          | `string?` | Base URL used by `url()` to build public file URLs.                               |
| `forcePathStyle`   | `boolean?`| Force path-style URLs (e.g. `https://host/bucket/key`). Required for R2 and MinIO. |

## `s3(config)`

`s3(config)` returns a `StorageAdapterProvider` that registers the S3 adapter under the `'s3'` driver name.

```ts
import { s3 } from '@rudderjs/storage'

// Returned provider is registered automatically via @rudderjs/storage dynamic loading.
// You do not need to add it to bootstrap/providers.ts manually.
const provider = s3(diskConfig)
```

## Compatible Services

This adapter works with any S3-compatible object storage service:

| Service                  | Notes                                         |
|--------------------------|-----------------------------------------------|
| AWS S3                   | Set `region`; no `endpoint` or `forcePathStyle` needed. |
| Cloudflare R2            | Set `endpoint` + `forcePathStyle: true`.      |
| MinIO                    | Set `endpoint` + `forcePathStyle: true`.      |
| DigitalOcean Spaces      | Set `endpoint` to your Spaces endpoint URL.   |

## Cloudflare R2 Example

```ts
// config/storage.ts
r2: {
  driver: 's3',
  bucket: Env.get('R2_BUCKET'),
  endpoint: Env.get('R2_ENDPOINT'), // https://<account-id>.r2.cloudflarestorage.com
  accessKeyId: Env.get('R2_ACCESS_KEY_ID'),
  secretAccessKey: Env.get('R2_SECRET_ACCESS_KEY'),
  baseUrl: Env.get('R2_PUBLIC_URL'),  // your public R2 custom domain
  forcePathStyle: true,
}
```

## Notes

- The underlying `@aws-sdk/client-s3` SDK is an optional peer dependency — you must install it explicitly.
- `accessKeyId` and `secretAccessKey` fall back to the `AWS_ACCESS_KEY_ID` and `AWS_SECRET_ACCESS_KEY` environment variables when not specified in config.
- Set `forcePathStyle: true` when using Cloudflare R2 or MinIO — these services do not support virtual-hosted-style URLs.
- `baseUrl` is used by `Storage.url(path)` to construct public-facing URLs. If omitted, a pre-signed URL is generated instead.
