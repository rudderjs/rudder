# @rudderjs/server-hono

Hono.js server adapter — implements `ServerAdapter` from `@rudderjs/contracts`.

Normalizes Hono requests/responses, handles Vike SSR integration, WebSocket upgrade patching, and ViewResponse detection (duck-typed via `__rudder_view__`).

Peer of `@rudderjs/core` — never add core to `dependencies` (same cycle rule as router).

`req.ip` is set by `extractIp(c, trustProxy)` in `normalizeRequest()`. `trustProxy` is `boolean | number`: `false` = socket only; `true` = trust ONE proxy hop → `req.ip` is the **rightmost** `x-forwarded-for` entry (the address the immediately-trusted proxy appended, which a client can't forge); `number N` = trust N chained hops → the Nth entry from the right (`parts[len - N]`). **Take the rightmost, never the leftmost** — the leftmost is whatever the client sent and is spoofable whenever the proxy appends rather than replaces the header (the nginx `proxy_add_x_forwarded_for` default), which would defeat ip-keyed rate limits / allowlists. `x-real-ip` is the fallback after `x-forwarded-for` when trusted. The direct socket address is the universal fallback (srvx's `request.ip`/`runtime.node` in the vike prod server, `env.incoming` under `@hono/node-server`), plus a dev-only `x-real-ip` stand-in injected by the `rudderjs:ip` Vite plugin (gated off `NODE_ENV=production` — the vite pipeline hands the adapter a plain web Request with no socket). Client-sent proxy headers are never read when `trustProxy` is off. Normalizes `::1` → `127.0.0.1` and strips `::ffff:` IPv4-mapping. Secure-by-default vs Laravel's `TrustProxies = '*'` (which trusts the whole client-supplied chain).

**Request body cap**: `readBodyText` bounds the JSON / form-urlencoded body to `HonoConfig.bodyLimit` (default 1 MB) — a declared `Content-Length` over the limit is rejected BEFORE buffering, and a body with no/lying length is buffered once then rejected if oversized; either way `PayloadTooLargeError` (HTTP 413). Multipart is NOT capped here (handlers stream it).

**Dev error page is secure-by-default**: gated by `devErrorPageEnabled(process.env)` — the rich page (full stack + on-disk source + all request headers) renders ONLY when the env is explicitly `local`/`development`/`dev`; an unset or unknown env is treated as production so a deploy that forgets `NODE_ENV=production` can't leak source + secret headers.

**Internal SPA-nav URL** (`.pageContext.json` rewrite → ViewResponse `toResponse`) travels via a per-request AsyncLocalStorage (`spaNavUrlStore`), NOT a client-visible header. Do not reintroduce an `x-rudder-original-url` request header — a direct client could forge it to inject an arbitrary URL into Vike's `renderPage` routing. Handlers/middleware read the same signal via `req.spaNavUrl` (the original `/<path>/index.pageContext.json` URL when the adapter rewrote this request into a controller-view call, else `undefined`) and the boolean `req.isPageContextRequest`. Both are getters over `spaNavUrlStore.getStore()` defined in `normalizeRequest()` alongside `req.session`/`req.user`/`req.token`, so they stay unforgeable (`undefined`/`false` on any direct request). They're the supported replacement for the removed `x-rudder-original-url` header — guard/policy middleware uses them to return a Vike-parseable JSON envelope for SPA fetches vs a real `302`/HTML for top-level navs.

WebSocket upgrade must be patched at module load time, not lazily.

## Response header handling

`normalizeResponse` tracks Set-Cookie separately from other headers (as an array, not a record) so cooperative cookie-writing middleware — CsrfMiddleware + SessionMiddleware is the canonical pair — don't clobber each other. When the handler returns a `ViewResponse` or raw `Response`, `c.res` is set directly bypassing `res.json()/res.send()`; the route handler then calls a stashed `__rjs_merge_pending` function to append the wrapper's pending headers to `c.res.headers` in place. **Mutate, don't clone**: `new Response(body, { headers: someHeaders })` collapses multi-value Set-Cookie in Node's undici-backed fetch.

## Dev error page (`error-page.ts`)

Ignition-style HTML rendered for uncaught errors in `APP_ENV=local|dev`. Renders parsed stack frames, source-code context (with `resolveErrorLine` working around tsx / Vite-SSR's inaccurate line numbers), and a Copy-as-Markdown button.

**`RUDDERJS` version badge** — `resolveRudderVersion(fromDir = cwd)` resolves the app's installed `@rudderjs/core` version via `createRequire(cwd)` (mirrors `@rudderjs/vite`'s helper; cwd is the app root at request time), falling back to this adapter's own version then `null` (badge omitted). Do NOT go back to reading this package's own `package.json` via `import.meta.url` — that reported the *adapter* version mislabeled as "RudderJS" and leaked a hard-coded `1.x` placeholder on bundled/serverless deploys. Test gotcha: assert on the rendered badge fragment `badge-gray">RUDDERJS 1.x` (the `">` survives), never the bare `RUDDERJS 1.x` — the HTML-escaped source-context echo of the test file false-positives the bare form.

**Editor-launch on stack frames** — frames are wrapped in `<a href="vscode://...">` anchors driven by the `APP_EDITOR` env var (default `vscode`; supports `cursor`, `webstorm`, `phpstorm`, `idea`, `sublime`, `atom`, `none`). Helper lives at `src/editor-launch.ts` — keep the per-editor URL templates there. Unknown values fall back to `vscode` with a single dev-time warning (`_warnedUnknown` is process-scoped). Windows paths are forward-slashed before being embedded in the URL — same portability fix as the dynamic-import pathToFileURL pattern.

**Primary "Open in editor" action** — a prominent button in the title-row (next to Copy-as-Markdown) that opens the TOP application frame (`topFrame`) in the resolved editor, reusing `buildEditorUrl()` (no new scheme logic). Rendered only when `topFrameEditorUrl` is non-null — omitted when `APP_EDITOR=none` or the error has no stack. Testing note: assert on the distinctive rendered anchor `<a class="action-btn open-editor-btn" href="`, NOT the bare `open-editor-btn` class — the latter also appears in the `<style>` block and in the HTML-escaped source-context echo of the test file itself.
